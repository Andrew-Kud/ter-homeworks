# Удаление - восстановление .tfstate:

1.
terraform state list
- Запиши id

2.

Запиши id сети и подсети:

terraform state show module.vpc_dev.yandex_vpc_network.network
terraform state show module.vpc_dev.yandex_vpc_network.network | grep id

terraform state show module.vpc_dev.yandex_vpc_subnet.subnet
terraform state show module.vpc_dev.yandex_vpc_subnet.subnet | grep id



Удаляй в порядке зависимости - подсеть - сеть:

удалить подсеть из стейта (в облаке останется):
terraform state rm module.vpc_dev.yandex_vpc_subnet.subnet

удалить сеть из стейта (в облаке останется):
terraform state rm module.vpc_dev.yandex_vpc_network.network


3.

Запиши id ВМ (тот, что над platform_id и под platform_id):

terraform state show 'module.test-vm-1.yandex_compute_instance.vm[0]'
terraform state show 'module.test-vm-1.yandex_compute_instance.vm[0]' | grep id

terraform state show 'module.test-vm-2.yandex_compute_instance.vm[0]'
terraform state show 'module.test-vm-2.yandex_compute_instance.vm[0]' | grep id

Можно сохранить стейт конкретных ресурсов:
terraform state show 'module.test-vm-1.yandex_compute_instance.vm[0]' > vm1-state-before-rm.txt
terraform state show 'module.test-vm-2.yandex_compute_instance.vm[0]' > vm2-state-before-rm.txt


удалить ВМ из стейта (в облаке останется):

terraform state rm module.test-vm-1.yandex_compute_instance.vm[0]
terraform state rm module.test-vm-2.yandex_compute_instance.vm[0]


Останотся только data-источники:
terraform state list
# module.test-vm-1.data.yandex_compute_image.my_image
# module.test-vm-2.data.yandex_compute_image.my_image


4.
Восстановление сети - импортируй в порядке зависимости - сеть - подсеть.

id сети:
terraform import module.vpc_dev.yandex_vpc_network.network enp74vas1fa1kir4b6nk

id подсети:
